Quasar's CPI uses two types: `CpiCall` for fixed-size instruction data and `BufCpiCall` for variable-length data. Both are stack-allocated with no heap allocation.

## CpiCall and BufCpiCall

`CpiCall<'a, ACCTS, DATA>` has const-generic account count and data size. `BufCpiCall<'a, ACCTS, MAX>` has a compile-time maximum data capacity but tracks actual length at runtime.

Both support three invocation methods:

```rust
cpi_call.invoke()?;                            // No PDA signers
cpi_call.invoke_signed(&seeds)?;               // Single PDA signer
cpi_call.invoke_with_signers(&[s1, s2])?;      // Multiple PDA signers
```

## System Program CPI

Available as free functions in `quasar_lang::cpi::system` or as methods on `Program<System>`:

| Function | Return Type | Description |
|----------|-------------|-------------|
| `transfer` | `CpiCall<2, 12>` | Transfer lamports (discriminator `2`) |
| `create_account` | `CpiCall<2, 52>` | Create account with space and owner (discriminator `0`) |
| `assign` | `CpiCall<1, 36>` | Reassign account to a new owner (discriminator `1`) |
| `create_account_with_minimum_balance` | `Result<CpiCall<2, 52>, ProgramError>` | Create with rent-exempt balance (fetches Rent sysvar if not provided) |

Free functions take `&AccountView` references:

```rust
use quasar_lang::cpi::system;

system::transfer(from_view, to_view, 1_000_000).invoke()?;
system::create_account(payer_view, new_view, lamports, space, &owner).invoke()?;
system::assign(account_view, &new_owner).invoke()?;
```

`Program<System>` methods accept `&impl AsAccountView`, so you can pass typed accounts directly:

```rust
self.system_program
    .transfer(self.user, self.vault, amount)
    .invoke()?;

self.system_program
    .create_account_with_minimum_balance(
        self.payer,
        self.new_account,
        space,
        &owner,
        Some(&*self.rent),  // Option<&Rent>
    )?
    .invoke()?;
```

## PDA Signing with Bumps

Use the `Bumps` struct from `#[derive(Accounts)]`. Its `*_seeds()` methods return `[Seed; N]` arrays for `invoke_signed`:

```rust
impl<'info> Take<'info> {
    pub fn withdraw_tokens_and_close(&mut self, bumps: &TakeBumps) -> Result<(), ProgramError> {
        let seeds = bumps.escrow_seeds();

        // Transfer tokens from vault to taker, signed by the escrow PDA
        self.token_program
            .transfer(
                self.vault_ta_a,
                self.taker_ta_a,
                self.escrow,
                self.vault_ta_a.amount(),
            )
            .invoke_signed(&seeds)?;

        // Close the vault token account, signed by the escrow PDA
        self.vault_ta_a
            .close(self.token_program, self.taker, self.escrow)
            .invoke_signed(&seeds)
    }
}
```

See [Accounts and Validation](./accounts-and-validation) for the full `Bumps` struct reference.

## Token Program CPI

The `quasar-spl` crate provides typed CPI helpers on `Program<Token>`:

```rust
// Transfer tokens (no PDA signer)
self.token_program
    .transfer(self.taker_ta_b, self.maker_ta_b, self.taker, amount)
    .invoke()?;

// Transfer tokens (PDA signer)
self.token_program
    .transfer(self.vault_ta_a, self.taker_ta_a, self.escrow, amount)
    .invoke_signed(&seeds)?;
```

Token account `close`:

```rust
self.vault_ta_a
    .close(self.token_program, self.destination, self.authority)
    .invoke_signed(&seeds)?;
```

## declare_program! — Typed CPI from IDL

`declare_program!` reads an IDL JSON at compile time and generates typed CPI functions:

```rust
declare_program!(my_program, "path/to/program.idl.json");
```

This generates:
- `pub const ID: Address` -- the program address from the IDL
- A program type (e.g., `MyProgram`) with executable and address checks
- A free function per instruction (takes `&AccountView`, returns `CpiCall`)
- Methods on the program type (takes `&impl AsAccountView`)

### Example

For an IDL instruction `"Transfer"` with discriminator `[2]`, two accounts (`from`: writable+signer, `to`: writable), and one arg (`amount: u64`):

```rust
pub mod my_program {
    pub const ID: Address = address!("...");

    // Free function: accounts as &AccountView
    pub fn transfer<'a>(
        __program: &'a AccountView,
        from: &'a AccountView,
        to: &'a AccountView,
        amount: u64,
    ) -> CpiCall<'a, 2, 9> { /* ... */ }

    // Method: accounts as &impl AsAccountView
    impl MyProgram {
        pub fn transfer<'a>(
            &'a self,
            from: &'a impl AsAccountView,
            to: &'a impl AsAccountView,
            amount: u64,
        ) -> CpiCall<'a, 2, 9> { /* ... */ }
    }
}
```

The data buffer size (`9` = 1-byte discriminator + 8-byte u64) is computed at compile time. Account flags (`writable`, `signer`) come from IDL metadata.

### Supported Argument Types

| IDL Type | Rust Type | Size |
|----------|-----------|------|
| `u8`, `i8`, `bool` | `u8`, `i8`, `bool` | 1 byte |
| `u16`, `i16` | `u16`, `i16` | 2 bytes |
| `u32`, `i32` | `u32`, `i32` | 4 bytes |
| `u64`, `i64` | `u64`, `i64` | 8 bytes |
| `u128`, `i128` | `u128`, `i128` | 16 bytes |
| `pubkey` | `&Address` | 32 bytes |

Struct types are not supported in CPI arguments -- only primitives and addresses.

## Events

### `emit!` (~100 CU)

Writes event data to the program log:

```rust
emit!(MakeEvent {
    escrow: *self.escrow.address(),
    maker: *self.maker.address(),
    mint_a: *self.mint_a.address(),
    mint_b: *self.mint_b.address(),
    deposit,
    receive,
});
```

Fast but spoofable -- any program can write to the log.

### `emit_cpi!` (~1,000 CU)

Requires `program` and `event_authority` fields in the accounts struct. The program invokes itself with event data prefixed by `0xFF`, signed by the `EventAuthority` PDA (seeds: `["__event_authority"]`). Clients can verify the event originated from your program because its ID appears in the transaction trace.
