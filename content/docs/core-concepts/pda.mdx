Quasar provides optimized PDA derivation and a declarative syntax for seed validation, bump discovery, and PDA signing in CPI.

## Seeds in Account Validation

### Verifying an Existing PDA

When the bump is stored in the account data, use `bump = expr` to verify the address:

```rust
#[derive(Accounts)]
pub struct Take<'info> {
    #[account(
        has_one = maker,
        close = taker,
        seeds = [b"escrow", maker],
        bump = escrow.bump
    )]
    pub escrow: &'info mut Account<Escrow>,
    pub taker: &'info mut Signer,
    pub maker: &'info mut UncheckedAccount,
}
```

With `bump = escrow.bump`, the generated code calls `verify_program_address`: one SHA-256 hash, one comparison. This is the cheapest path.

### Discovering a Bump

When initializing a new PDA, use `bump` without a value to search for the canonical bump:

```rust
#[derive(Accounts)]
pub struct Make<'info> {
    pub maker: &'info mut Signer,
    #[account(init, payer = maker, seeds = [b"escrow", maker], bump)]
    pub escrow: &'info mut Account<Escrow>,
}
```

The generated code iterates from bump 255 downward until it finds a valid PDA. The discovered bump is stored in the `Bumps` struct.

### Seed Expressions

Seeds can be:

- **Byte literals**: `b"escrow"`, `b"vault"`
- **Account field references**: `maker` (automatically converted to the 32-byte address)
- **Arbitrary expressions**: any expression that evaluates to `&[u8]`

## The Bumps Struct

For each `#[derive(Accounts)]` struct that contains PDA-validated fields, the macro generates a companion `Bumps` struct:

```rust
#[derive(Copy, Clone)]
pub struct MakeBumps {
    pub escrow: u8,
}
```

Only fields with `seeds` + `bump` get an entry. The bump value is populated during parsing and made available via `ctx.bumps`.

### Seed Helper Methods

The bumps struct generates `*_seeds()` methods that return the complete seed array for CPI signing:

```rust
impl TakeBumps {
    pub fn escrow_seeds(&self) -> [Seed; 3] {
        // [b"escrow", maker_address, &[self.escrow]]
    }
}
```

Usage:

```rust
pub fn withdraw_tokens_and_close(&mut self, bumps: &TakeBumps) -> Result<(), ProgramError> {
    let seeds = bumps.escrow_seeds();
    self.token_program
        .transfer(self.vault_ta_a, self.taker_ta_a, self.escrow, amount)
        .invoke_signed(&seeds)
}
```

## PDA Signing in CPI

When your program needs to "sign" as a PDA in a cross-program invocation, pass the PDA's seeds to `invoke_signed`. The runtime verifies that seeds + program ID produce the signer address.

The typical flow:

1. During account parsing, the bump is discovered or verified and stored in `ctx.bumps`.
2. In the handler, call `bumps.field_seeds()` to get the seed array.
3. Pass the seeds to `.invoke_signed(&seeds)` on a CPI call.

See [Cross-Program Invocations](./cpi) for the full CPI API.

## Storing Bumps

Store the bump in the account data so future instructions use `bump = account.bump` (verify) instead of re-deriving (find):

```rust
#[account(discriminator = 1)]
pub struct Escrow {
    pub maker: Address,
    pub mint_a: Address,
    pub mint_b: Address,
    pub maker_ta_b: Address,
    pub receive: u64,
    pub bump: u8,
}
```

During initialization:

```rust
pub fn make_escrow(&mut self, receive: u64, bumps: &MakeBumps) -> Result<(), ProgramError> {
    self.escrow.set_inner(
        *self.maker.address(),
        *self.mint_a.address(),
        *self.mint_b.address(),
        *self.maker_ta_b.address(),
        receive,
        bumps.escrow,  // Store the bump
    );
    Ok(())
}
```

Verify is cheaper than find (one hash vs. iterating), and storing the canonical bump prevents accidental use of non-canonical bumps.

## PDA Address Limits

The Solana runtime enforces a maximum of 16 seeds per PDA (plus the bump seed, for 17 total). Each individual seed can be at most 32 bytes.
