title: Mollusk

[Mollusk](https://github.com/buffalojoec/mollusk) is a lightweight SVM test harness that processes individual instructions in isolation, without running a validator.

## Setup

Add these dev-dependencies to your `Cargo.toml`:

```toml
[dev-dependencies]
mollusk-svm = "0.10.3"
solana-account = { version = "3.4.0" }
solana-address = { version = "2.2.0", features = ["decode"] }
solana-instruction = { version = "3.2.0", features = ["bincode"] }
```

Or run `quasar init` and select **Mollusk** when prompted for a testing framework.

For instruction building, you can use either:
- The inline `client` module generated by `#[program]` (import as `crate::client::*` or `my_program::client::*`), or
- The standalone client crate generated by `quasar idl` at `target/client/rust/<name>-client/`.

The inline module requires no extra dependency. For the standalone crate:

```toml
[dev-dependencies]
my-program-client = { path = "target/client/rust/my-program-client" }
```

## Creating a Mollusk instance

Point Mollusk at your program ID and compiled `.so` binary:

```rust
use mollusk_svm::Mollusk;

fn setup() -> Mollusk {
    Mollusk::new(&crate::ID, "../../target/deploy/my_program")
}
```

The second argument is the path to the `.so` file, without the extension.

## Processing instructions

Pass an `Instruction` and a slice of `(Address, Account)` tuples to `process_instruction`:

```rust
use mollusk_svm::{result::ProgramResult as MolluskResult, Mollusk};
use solana_account::Account;
use solana_address::Address;
use solana_instruction::Instruction;

// Import from the inline client module
use my_program::client::InitializeInstruction;

#[test]
fn test_initialize() {
    let mollusk = setup();

    let payer = Address::new_unique();
    let payer_account = Account::new(10_000_000_000, 0, &Address::default());

    let instruction: Instruction = InitializeInstruction {
        payer,
        system_program: Address::default(),
    }
    .into();

    let result = mollusk.process_instruction(
        &instruction,
        &[
            (payer, payer_account),
            (Address::default(), Account::new(1, 0, &Address::default())),
        ],
    );

    assert!(result.program_result.is_ok());
}
```

The generated client types implement `Into<Instruction>`, setting the discriminator, account metas, and serialized instruction data.

## Asserting on results

`process_instruction` returns:

- **`program_result`** -- `MolluskResult::Success` or `MolluskResult::Failure(ProgramError)`
- **`resulting_accounts`** -- Updated `(Address, Account)` tuples after execution
- **`compute_units_consumed`** -- Number of compute units used

### Checking success

```rust
assert!(result.program_result.is_ok());
```

With a message on failure:

```rust
assert!(
    result.program_result.is_ok(),
    "deposit failed: {:?}",
    result.program_result
);
```

### Checking specific errors

```rust
use mollusk_svm::result::ProgramResult as MolluskResult;
use quasar_lang::prelude::ProgramError;

assert_eq!(
    result.program_result,
    MolluskResult::Failure(ProgramError::IllegalOwner)
);
```

Common Quasar error codes:

| Error | Code | Meaning |
|-------|------|---------|
| `ProgramError::IllegalOwner` | -- | Account not owned by expected program |
| `ProgramError::InvalidAccountData` | -- | Wrong discriminator, not executable, or all-zero discriminator |
| `ProgramError::AccountDataTooSmall` | -- | Account data shorter than expected |
| `ProgramError::MissingRequiredSignature` | -- | Signer flag not set |
| `ProgramError::Immutable` | -- | Writable flag not set on a `mut` account |
| `ProgramError::IncorrectProgramId` | -- | Program account address does not match expected ID |
| `ProgramError::AccountBorrowFailed` | -- | Duplicate accounts where not allowed |
| `ProgramError::Custom(3004)` | 3004 | `QuasarError::ConstraintViolation` |
| `ProgramError::Custom(3005)` | 3005 | `QuasarError::HasOneMismatch` |
| `ProgramError::Custom(3012)` | 3012 | `QuasarError::AddressMismatch` |

### Inspecting account state

Read modified account data after execution:

```rust
let result = mollusk.process_instruction(&instruction, &accounts);
assert!(result.program_result.is_ok());

let data = &result.resulting_accounts[0].1.data;
let value = u64::from_le_bytes(data[33..41].try_into().unwrap());
assert_eq!(value, 999);
```

### Checking compute units

```rust
println!("CU consumed: {}", result.compute_units_consumed);
assert!(result.compute_units_consumed < 5_000);
```

## Multi-step testing

Feed resulting accounts from one instruction into the next:

```rust
#[test]
fn test_deposit_then_withdraw() {
    let mollusk = setup();
    let (system_program, system_program_account) =
        mollusk_svm::program::keyed_account_for_system_program();

    let user = Address::new_unique();
    let user_account = Account::new(10_000_000_000, 0, &system_program);

    let (vault, _vault_bump) =
        Address::find_program_address(&[b"vault", user.as_ref()], &crate::ID);
    let vault_account = Account::new(0, 0, &crate::ID);

    let deposit_amount: u64 = 1_000_000_000;

    // Step 1: Deposit
    let deposit_ix: Instruction = DepositInstruction {
        user, vault, system_program, amount: deposit_amount,
    }.into();

    let result = mollusk.process_instruction(
        &deposit_ix,
        &[
            (user, user_account),
            (vault, vault_account),
            (system_program, system_program_account),
        ],
    );
    assert!(result.program_result.is_ok(), "deposit failed: {:?}", result.program_result);

    let user_after_deposit = result.resulting_accounts[0].1.clone();
    let vault_after_deposit = result.resulting_accounts[1].1.clone();

    // Step 2: Withdraw using updated state
    let withdraw_amount: u64 = 500_000_000;

    let withdraw_ix: Instruction = WithdrawInstruction {
        user, vault, amount: withdraw_amount,
    }.into();

    let result = mollusk.process_instruction(
        &withdraw_ix,
        &[(user, user_after_deposit.clone()), (vault, vault_after_deposit)],
    );
    assert!(result.program_result.is_ok(), "withdraw failed: {:?}", result.program_result);

    let user_final = result.resulting_accounts[0].1.lamports;
    let vault_final = result.resulting_accounts[1].1.lamports;

    assert_eq!(user_final, user_after_deposit.lamports + withdraw_amount);
    assert_eq!(vault_final, deposit_amount - withdraw_amount);
}
```

## Building account data manually

Construct raw account data to test validation:

```rust
use {
    mollusk_svm::{result::ProgramResult as MolluskResult, Mollusk},
    quasar_lang::prelude::ProgramError,
    my_program::client::*,
    solana_account::Account,
    solana_address::Address,
    solana_instruction::Instruction,
};

const ACCOUNT_SIZE: usize = 41; // 1 discriminator + 32 authority + 8 value

fn build_valid_account_data(authority: Address, value: u64) -> Vec<u8> {
    let mut data = vec![0u8; ACCOUNT_SIZE];
    data[0] = 1; // discriminator
    data[1..33].copy_from_slice(authority.as_ref());
    data[33..41].copy_from_slice(&value.to_le_bytes());
    data
}

#[test]
fn test_wrong_owner_rejected() {
    let mollusk = setup();
    let account_addr = Address::new_unique();
    let wrong_owner = Address::new_unique();
    let data = build_valid_account_data(Address::new_unique(), 42);

    let instruction: Instruction = AccountCheckInstruction {
        account: account_addr,
    }.into();

    let result = mollusk.process_instruction(
        &instruction,
        &[(
            account_addr,
            Account {
                lamports: 1_000_000,
                data,
                owner: wrong_owner, // Not the program -- triggers IllegalOwner
                executable: false,
                rent_epoch: 0,
            },
        )],
    );

    assert_eq!(
        result.program_result,
        MolluskResult::Failure(ProgramError::IllegalOwner)
    );
}
```

## When to use Mollusk vs QuasarSVM

| | Mollusk | QuasarSVM |
|---|---------|-----------|
| **Scope** | Single instruction | Full transactions |
| **Speed** | Fastest | Fast |
| **API** | `process_instruction` | `process_transaction` |
| **Best for** | Unit tests, validation checks, error paths | Integration tests, multi-instruction flows |
| **System programs** | Manual setup with `keyed_account_for_system_program()` | `.addSystemProgram()` builder |

Use Mollusk for unit-testing individual instructions. Use QuasarSVM for transaction-level integration tests and multi-instruction flows.

Both harnesses are available from Rust. For TypeScript, use the QuasarSVM TypeScript bindings.
